9 ## Overview
10 
11 Things to know when using the popover plugin:
12 
13 - Popovers rely on the 3rd party library [Popper.js](https://popper.js.org/) for positioning. You must include [popper.min.js]({{ site.cdn.popper }}) before bootstrap.js or use `bootstrap.bundle.min.js` / `bootstrap.bundle.js` which contains Popper.js in order for popovers to work!
14 - Popovers require the [tooltip plugin]({{ site.baseurl }}/docs/{{ site.docs_version }}/components/tooltips/) as a dependency.
15 - If you're building our JavaScript from source, it [requires `util.js`]({{ site.baseurl }}/docs/{{ site.docs_version }}/getting-started/javascript/#util).
16 - Popovers are opt-in for performance reasons, so **you must initialize them yourself**.
17 - Zero-length `title` and `content` values will never show a popover.
18 - Specify `container: 'body'` to avoid rendering problems in more complex components (like our input groups, button groups, etc).
19 - Triggering popovers on hidden elements will not work.
20 - Popovers for `.disabled` or `disabled` elements must be triggered on a wrapper element.
21 - When triggered from anchors that wrap across multiple lines, popovers will be centered between the anchors' overall width. Use `white-space: nowrap;` on your `<a>`s to avoid this behavior.
22 - Popovers must be hidden before their corresponding elements have been removed from the DOM.
23 
24 Keep reading to see how popovers work with some examples.
25 
26 ## Example: Enable popovers everywhere
27 
28 One way to initialize all popovers on a page would be to select them by their `data-toggle` attribute:
29 
30 {% highlight js %}
31 $(function () {
32  $('[data-toggle="popover"]').popover()
33 })
34 {% endhighlight %}
35 
36 ## Example: Using the `container` option
37 
38 When you have some styles on a parent element that interfere with a popover, you'll want to specify a custom `container` so that the popover's HTML appears within that element instead.
39 
40 {% highlight js %}
41 $(function () {
42  $('.example-popover').popover({
43  container: 'body'
44  })
45 })
46 {% endhighlight %}

94 ### Dismiss on next click
95 
96 Use the `focus` trigger to dismiss popovers on the user's next click of a different element than the toggle element.
97 
98 {% callout danger %}
99 #### Specific markup required for dismiss-on-next-click
100 
101 For proper cross-browser and cross-platform behavior, you must use the `<a>` tag, _not_ the `<button>` tag, and you also must include a [`tabindex`](https://developer.mozilla.org/en-US/docs/Web/HTML/Global_attributes/tabindex) attribute.
102 {% endcallout %}
103 
104 {% example html %}
105 <a tabindex="0" class="btn btn-lg btn-danger" role="button" data-toggle="popover" data-trigger="focus" title="Dismissible popover" data-content="And here's some amazing content. It's very engaging. Right?">Dismissible popover</a>
106 {% endexample %}
107 
108 {% highlight js %}
109 $('.popover-dismiss').popover({
110  trigger: 'focus'
111 })
112 {% endhighlight %}

114 ### Disabled elements
115 
116 Elements with the `disabled` attribute aren't interactive, meaning users cannot hover or click them to trigger a popover (or tooltip). As a workaround, you'll want to trigger the popover from a wrapper `<div>` or `<span>` and override the `pointer-events` on the disabled element.
117 
118 For disabled popover triggers, you may also prefer `data-trigger="hover"` so that the popover appears as immediate visual feedback to your users as they may not expect to _click_ on a disabled element.
119 
120 {% example html %}
121 <span class="d-inline-block" data-toggle="popover" data-content="Disabled popover">
122  <button class="btn btn-primary" style="pointer-events: none;" type="button" disabled>Disabled button</button>
123 </span>
124 {% endexample %}

260 #### `$().popover(options)`
261 
262 Initializes popovers for an element collection.
263 
264 #### `.popover('show')`
265 
266 Reveals an element's popover. **Returns to the caller before the popover has actually been shown** (i.e. before the `shown.bs.popover` event occurs). This is considered a "manual" triggering of the popover. Popovers whose both title and content are zero-length are never displayed.
267 
268 {% highlight js %}$('#element').popover('show'){% endhighlight %}
269 
270 #### `.popover('hide')`
271 
272 Hides an element's popover. **Returns to the caller before the popover has actually been hidden** (i.e. before the `hidden.bs.popover` event occurs). This is considered a "manual" triggering of the popover.
273 
274 {% highlight js %}$('#element').popover('hide'){% endhighlight %}
275 
276 #### `.popover('toggle')`
277 
278 Toggles an element's popover. **Returns to the caller before the popover has actually been shown or hidden** (i.e. before the `shown.bs.popover` or `hidden.bs.popover` event occurs). This is considered a "manual" triggering of the popover.
279 
280 {% highlight js %}$('#element').popover('toggle'){% endhighlight %}
281 
282 #### `.popover('dispose')`
283 
284 Hides and destroys an element's popover. Popovers that use delegation (which are created using [the `selector` option](#options)) cannot be individually destroyed on descendant trigger elements.
285 
286 {% highlight js %}$('#element').popover('dispose'){% endhighlight %}
287 
288 #### `.popover('enable')`
289 
290 Gives an element's popover the ability to be shown. **Popovers are enabled by default.**
291 
292 {% highlight js %}$('#element').popover('enable'){% endhighlight %}
293 
294 #### `.popover('disable')`
295 
296 Removes the ability for an element's popover to be shown. The popover will only be able to be shown if it is re-enabled.
297 
298 {% highlight js %}$('#element').popover('disable'){% endhighlight %}
299 
300 #### `.popover('toggleEnabled')`
301 
302 Toggles the ability for an element's popover to be shown or hidden.
303 
304 {% highlight js %}$('#element').popover('toggleEnabled'){% endhighlight %}
305 
306 #### `.popover('update')`
307 
308 Updates the position of an element's popover.
309 
310 {% highlight js %}$('#element').popover('update'){% endhighlight %}
