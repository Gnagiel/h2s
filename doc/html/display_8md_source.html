9 ## How it works
10 
11 Change the value of the [`display` property](https://developer.mozilla.org/en-US/docs/Web/CSS/display) with our responsive display utility classes. We purposely support only a subset of all possible values for `display`. Classes can be combined for various effects as you need.
12 
13 ## Notation
14 
15 Display utility classes that apply to all [breakpoints]({{ site.baseurl }}/docs/{{ site.docs_version }}/layout/overview/#responsive-breakpoints), from `xs` to `xl`, have no breakpoint abbreviation in them. This is because those classes are applied from `min-width: 0;` and up, and thus are not bound by a media query. The remaining breakpoints, however, do include a breakpoint abbreviation.
16 
17 As such, the classes are named using the format:
18 
19 * `.d-{value}` for `xs`
20 * `.d-{breakpoint}-{value}` for `sm`, `md`, `lg`, and `xl`.
21 
22 Where *value* is one of:
23 
24 * `none`
25 * `inline`
26 * `inline-block`
27 * `block`
28 * `table`
29 * `table-cell`
30 * `table-row`
31 * `flex`
32 * `inline-flex`
33 
34 The media queries effect screen widths with the given breakpoint *or larger*. For example, `.d-lg-none` sets `display: none;` on both `lg` and `xl` screens.
35 
36 ## Examples
37 
38 {% example html %}
39 <div class="d-inline p-2 bg-primary text-white">d-inline</div>
40 <div class="d-inline p-2 bg-dark text-white">d-inline</div>
41 {% endexample %}
42 
43 {% example html %}
44 <span class="d-block p-2 bg-primary text-white">d-block</span>
45 <span class="d-block p-2 bg-dark text-white">d-block</span>
46 {% endexample %}

48 ## Hiding elements
49 
50 For faster mobile-friendly development, use responsive display classes for showing and hiding elements by device. Avoid creating entirely different versions of the same site, instead hide element responsively for each screen size.
51 
52 To hide elements simply use the `.d-none` class or one of the `.d-{sm,md,lg,xl}-none` classes for any responsive screen variation.
53 
54 To show an element only on a given interval of screen sizes you can combine one `.d-*-none` class with a `.d-*-*` class, for example `.d-none .d-md-block .d-xl-none` will hide the element for all screen sizes except on medium and large devices.
55 
56 | Screen Size | Class |
57 | --- | --- |
58 | Hidden on all | `.d-none` |
59 | Hidden only on xs | `.d-none .d-sm-block` |
60 | Hidden only on sm | `.d-sm-none .d-md-block` |
61 | Hidden only on md | `.d-md-none .d-lg-block` |
62 | Hidden only on lg | `.d-lg-none .d-xl-block` |
63 | Hidden only on xl | `.d-xl-none` |
64 | Visible on all | `.d-block` |
65 | Visible only on xs | `.d-block .d-sm-none` |
66 | Visible only on sm | `.d-none .d-sm-block .d-md-none` |
67 | Visible only on md | `.d-none .d-md-block .d-lg-none` |
68 | Visible only on lg | `.d-none .d-lg-block .d-xl-none` |
69 | Visible only on xl | `.d-none .d-xl-block` |
70 
71 {% example html %}
72 <div class="d-lg-none">hide on screens wider than lg</div>
73 <div class="d-none d-lg-block">hide on screens smaller than lg</div>
74 {% endexample %}

76 ## Display in print
77 
78 Change the `display` value of elements when printing with our print display utility classes. Includes support for the same `display` values as our responsive `.d-*` utilities.
79 
80 - `.d-print-none`
81 - `.d-print-inline`
82 - `.d-print-inline-block`
83 - `.d-print-block`
84 - `.d-print-table`
85 - `.d-print-table-row`
86 - `.d-print-table-cell`
87 - `.d-print-flex`
88 - `.d-print-inline-flex`
89 
90 The print and display classes can be combined.
91 
92 {% example html %}
93 <div class="d-print-none">Screen Only (Hide on print only)</div>
94 <div class="d-none d-print-block">Print Only (Hide on screen only)</div>
95 <div class="d-none d-lg-block d-print-block">Hide up to large on screen, but always show on print</div>
96 {% endexample %}
