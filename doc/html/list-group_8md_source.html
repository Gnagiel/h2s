9 ## Basic example
10 
11 The most basic list group is an unordered list with list items and the proper classes. Build upon it with the options that follow, or with your own CSS as needed.
12 
13 {% example html %}
14 <ul class="list-group">
15  <li class="list-group-item">Cras justo odio</li>
16  <li class="list-group-item">Dapibus ac facilisis in</li>
17  <li class="list-group-item">Morbi leo risus</li>
18  <li class="list-group-item">Porta ac consectetur ac</li>
19  <li class="list-group-item">Vestibulum at eros</li>
20 </ul>
21 {% endexample %}

23 ## Active items
24 
25 Add `.active` to a `.list-group-item` to indicate the current active selection.
26

28 <ul class="list-group">
29  <li class="list-group-item active">Cras justo odio</li>
30  <li class="list-group-item">Dapibus ac facilisis in</li>
31  <li class="list-group-item">Morbi leo risus</li>
32  <li class="list-group-item">Porta ac consectetur ac</li>
33  <li class="list-group-item">Vestibulum at eros</li>
34 </ul>
35 {% endexample %}

37 ## Disabled items
38 
39 Add `.disabled` to a `.list-group-item` to make it _appear_ disabled. Note that some elements with `.disabled` will also require custom JavaScript to fully disable their click events (e.g., links).
40 
41 {% example html %}
42 <ul class="list-group">
43  <li class="list-group-item disabled">Cras justo odio</li>
44  <li class="list-group-item">Dapibus ac facilisis in</li>
45  <li class="list-group-item">Morbi leo risus</li>
46  <li class="list-group-item">Porta ac consectetur ac</li>
47  <li class="list-group-item">Vestibulum at eros</li>
48 </ul>
49 {% endexample %}

51 ## Links and buttons
52 
53 Use `<a>`s or `<button>`s to create _actionable_ list group items with hover, disabled, and active states by adding `.list-group-item-action`. We separate these pseudo-classes to ensure list groups made of non-interactive elements (like `<li>`s or `<div>`s) don't provide a click or tap affordance.
54 
55 Be sure to **not use the standard `.btn` classes here**.
56 
57 {% example html %}
58 <div class="list-group">
59  <a href="#" class="list-group-item list-group-item-action active">
60  Cras justo odio
61  </a>
62  <a href="#" class="list-group-item list-group-item-action">Dapibus ac facilisis in</a>
63  <a href="#" class="list-group-item list-group-item-action">Morbi leo risus</a>
64  <a href="#" class="list-group-item list-group-item-action">Porta ac consectetur ac</a>
65  <a href="#" class="list-group-item list-group-item-action disabled">Vestibulum at eros</a>
66 </div>
67 {% endexample %}
68 
69 With `<button>`s, you can also make use of the `disabled` attribute instead of the `.disabled` class. Sadly, `<a>`s don't support the disabled attribute.
70 
71 {% example html %}
72 <div class="list-group">
73  <button type="button" class="list-group-item list-group-item-action active">
74  Cras justo odio
75  </button>
76  <button type="button" class="list-group-item list-group-item-action">Dapibus ac facilisis in</button>
77  <button type="button" class="list-group-item list-group-item-action">Morbi leo risus</button>
78  <button type="button" class="list-group-item list-group-item-action">Porta ac consectetur ac</button>
79  <button type="button" class="list-group-item list-group-item-action" disabled>Vestibulum at eros</button>
80 </div>
81 {% endexample %}

97 ## Contextual classes
98 
99 Use contextual classes to style list items with a stateful background and color.
100 
101 {% example html %}
102 <ul class="list-group">
103  <li class="list-group-item">Dapibus ac facilisis in</li>
104 
105  {% for color in site.data.theme-colors %}
106  <li class="list-group-item list-group-item-{{ color.name }}">This is a {{ color.name }} list group item</li>{% endfor %}
107 </ul>
108 {% endexample %}
109 
110 Contextual classes also work with `.list-group-item-action`. Note the addition of the hover styles here not present in the previous example. Also supported is the `.active` state; apply it to indicate an active selection on a contextual list group item.
111 
112 {% example html %}
113 <div class="list-group">
114  <a href="#" class="list-group-item list-group-item-action">Dapibus ac facilisis in</a>
115 
116  {% for color in site.data.theme-colors %}
117  <a href="#" class="list-group-item list-group-item-action list-group-item-{{ color.name }}">This is a {{ color.name }} list group item</a>{% endfor %}
118 </div>
119 {% endexample %}

232 ### Using data attributes
233 
234 You can activate a list group navigation without writing any JavaScript by simply specifying `data-toggle="list"` or on an element. Use these data attributes on `.list-group-item`.
235 
236 <div role="tabpanel">
237 {% highlight html %}
238 <!-- List group -->
239 <div class="list-group" id="myList" role="tablist">
240  <a class="list-group-item list-group-item-action active" data-toggle="list" href="#home" role="tab">Home</a>
241  <a class="list-group-item list-group-item-action" data-toggle="list" href="#profile" role="tab">Profile</a>
242  <a class="list-group-item list-group-item-action" data-toggle="list" href="#messages" role="tab">Messages</a>
243  <a class="list-group-item list-group-item-action" data-toggle="list" href="#settings" role="tab">Settings</a>
244 </div>
245

247 <div class="tab-content">
248  <div class="tab-pane active" id="home" role="tabpanel">...</div>
249  <div class="tab-pane" id="profile" role="tabpanel">...</div>
250  <div class="tab-pane" id="messages" role="tabpanel">...</div>
251  <div class="tab-pane" id="settings" role="tabpanel">...</div>

256 ### Via JavaScript
257 
258 Enable tabbable list item via JavaScript (each list item needs to be activated individually):
259 
260 {% highlight js %}
261 $('#myList a').on('click', function (e) {
262  e.preventDefault()
263  $(this).tab('show')
264 })
265 {% endhighlight %}
266 
267 You can activate individual list item in several ways:
268 
269 {% highlight js %}
270 $('#myList a[href="#profile"]').tab('show') // Select tab by name
271 $('#myList a:first-child').tab('show') // Select first tab
272 $('#myList a:last-child').tab('show') // Select last tab
273 $('#myList a:nth-child(3)').tab('show') // Select third tab
274 {% endhighlight %}

317 #### .tab('show')
318 
319 Selects the given list item and shows its associated pane. Any other list item that was previously selected becomes unselected and its associated pane is hidden. **Returns to the caller before the tab pane has actually been shown** (for example, before the `shown.bs.tab` event occurs).
320 
321 {% highlight js %}
322 $('#someListItem').tab('show')
323 {% endhighlight %}

325 ### Events
326 
327 When showing a new tab, the events fire in the following order:
328 
329 1. `hide.bs.tab` (on the current active tab)
330 2. `show.bs.tab` (on the to-be-shown tab)
331 3. `hidden.bs.tab` (on the previous active tab, the same one as for the `hide.bs.tab` event)
332 4. `shown.bs.tab` (on the newly-active just-shown tab, the same one as for the `show.bs.tab` event)
333 
334 If no tab was already active, the `hide.bs.tab` and `hidden.bs.tab` events will not be fired.
335 
336 <table class="table table-bordered table-striped">
337  <thead>
338  <tr>
339  <th style="width: 150px;">Event type</th>
340  <th>Description</th>
341  </tr>
342  </thead>
343  <tbody>
344  <tr>
345  <td>show.bs.tab</td>
346  <td>This event fires on tab show, but before the new tab has been shown. Use <code>event.target</code> and <code>event.relatedTarget</code> to target the active tab and the previous active tab (if available) respectively.</td>
347  </tr>
348  <tr>
349  <td>shown.bs.tab</td>
350  <td>This event fires on tab show after a tab has been shown. Use <code>event.target</code> and <code>event.relatedTarget</code> to target the active tab and the previous active tab (if available) respectively.</td>
351  </tr>
352  <tr>
353  <td>hide.bs.tab</td>
354  <td>This event fires when a new tab is to be shown (and thus the previous active tab is to be hidden). Use <code>event.target</code> and <code>event.relatedTarget</code> to target the current active tab and the new soon-to-be-active tab, respectively.</td>
355  </tr>
356  <tr>
357  <td>hidden.bs.tab</td>
358  <td>This event fires after a new tab is shown (and thus the previous active tab is hidden). Use <code>event.target</code> and <code>event.relatedTarget</code> to target the previous active tab and the new active tab, respectively.</td>
359  </tr>
360  </tbody>
361 </table>
362 
363 {% highlight js %}
364 $('a[data-toggle="list"]').on('shown.bs.tab', function (e) {
365  e.target // newly activated tab
366  e.relatedTarget // previous active tab
367 })
368 {% endhighlight %}
