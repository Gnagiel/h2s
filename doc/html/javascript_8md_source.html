9 ## Individual or compiled
10 
11 Plugins can be included individually (using Bootstrap's individual `*.js` files), or all at once using `bootstrap.js` or the minified `bootstrap.min.js` (don't include both).
12 
13 ## Dependencies
14 
15 Some plugins and CSS components depend on other plugins. If you include plugins individually, make sure to check for these dependencies in the docs. Also note that **all plugins depend on jQuery** (this means jQuery must be included **before** the plugin files). [Consult our `package.json`]({{ site.repo }}/blob/v{{ site.current_version }}/package.json) to see which versions of jQuery are supported.
16 
17 Our dropdowns, popovers and tooltips also depend on [Popper.js](https://popper.js.org/).
18 
19 ## Data attributes
20 
21 Nearly all Bootstrap plugins can be enabled and configured through HTML alone with data attributes (our preferred way of using JavaScript functionality). Be sure to **only use one set of data attributes on a single element** (e.g., you cannot trigger a tooltip and modal from the same button.)
22 
23 However, in some situations it may be desirable to disable this functionality. To disable the data attribute API, unbind all events on the document namespaced with `data-api` like so:
24 
25 {% highlight js %}
26 $(document).off('.data-api')
27 {% endhighlight %}
28 
29 Alternatively, to target a specific plugin, just include the plugin's name as a namespace along with the data-api namespace like this:
30 
31 {% highlight js %}
32 $(document).off('.alert.data-api')
33 {% endhighlight %}
34 
35 ## Events
36 
37 Bootstrap provides custom events for most plugins' unique actions. Generally, these come in an infinitive and past participle form - where the infinitive (ex. `show`) is triggered at the start of an event, and its past participle form (ex. `shown`) is triggered on the completion of an action.
38 
39 All infinitive events provide [`preventDefault()`](https://developer.mozilla.org/en-US/docs/Web/API/Event/preventDefault) functionality. This provides the ability to stop the execution of an action before it starts. Returning false from an event handler will also automatically call `preventDefault()`.
40 
41 {% highlight js %}
42 $('#myModal').on('show.bs.modal', function (e) {
43  if (!data) return e.preventDefault() // stops modal from being shown
44 })
45 {% endhighlight %}
46 
47 ## Programmatic API
48 
49 We also believe you should be able to use all Bootstrap plugins purely through the JavaScript API. All public APIs are single, chainable methods, and return the collection acted upon.
50 
51 {% highlight js %}
52 $('.btn.danger').button('toggle').addClass('fat')
53 {% endhighlight %}
54 
55 All methods should accept an optional options object, a string which targets a particular method, or nothing (which initiates a plugin with default behavior):
56 
57 {% highlight js %}
58 $('#myModal').modal() // initialized with defaults
59 $('#myModal').modal({ keyboard: false }) // initialized with no keyboard
60 $('#myModal').modal('show') // initializes and invokes show immediately
61 {% endhighlight %}
62 
63 Each plugin also exposes its raw constructor on a `Constructor` property: `$.fn.popover.Constructor`. If you'd like to get a particular plugin instance, retrieve it directly from an element: `$('[rel="popover"]').data('popover')`.
64 
65 ### Asynchronous functions and transitions
66 
67 All programmatic API methods are **asynchronous** and returns to the caller once the transition is started but **before it ends**.
68 
69 In order to execute an action once the transition is complete, you can listen to the corresponding event.
70 
71 {% highlight js %}
72 $('#myCollapse').on('shown.bs.collapse', function (e) {
73  // Action to execute once the collapsible area is expanded
74 })
75 {% endhighlight %}
76 
77 In addition a method call on a **transitioning component will be ignored**.
78 
79 {% highlight js %}
80 $('#myCarousel').on('slid.bs.carousel', function (e) {
81  $('#myCarousel').carousel('2') // Will slide to the slide 2 as soon as the transition to slide 1 is finished
82 })
83 
84 $('#myCarousel').carousel('1') // Will start sliding to the slide 1 and returns to the caller
85 $('#myCarousel').carousel('2') // !! Will be ignored, as the transition to the slide 1 is not finished !!
86 {% endhighlight %}
87 
88 ### Default settings
89 
90 You can change the default settings for a plugin by modifying the plugin's `Constructor.Default` object:
91 
92 {% highlight js %}
93 $.fn.modal.Constructor.Default.keyboard = false // changes default for the modal plugin's `keyboard` option to false
94 {% endhighlight %}

96 ## No conflict
97 
98 Sometimes it is necessary to use Bootstrap plugins with other UI frameworks. In these circumstances, namespace collisions can occasionally occur. If this happens, you may call `.noConflict` on the plugin you wish to revert the value of.
99 
100 {% highlight js %}
101 var bootstrapButton = $.fn.button.noConflict() // return $.fn.button to previously assigned value
102 $.fn.bootstrapBtn = bootstrapButton // give $().bootstrapBtn the Bootstrap functionality
103 {% endhighlight %}
104 
105 ## Version numbers
106 
107 The version of each of Bootstrap's jQuery plugins can be accessed via the `VERSION` property of the plugin's constructor. For example, for the tooltip plugin:
108 
109 {% highlight js %}
110 $.fn.tooltip.Constructor.VERSION // => "{{ site.current_version }}"
111 {% endhighlight %}
112 
113 ## No special fallbacks when JavaScript is disabled
114 
115 Bootstrap's plugins don't fall back particularly gracefully when JavaScript is disabled. If you care about the user experience in this case, use [`<noscript>`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/noscript) to explain the situation (and how to re-enable JavaScript) to your users, and/or add your own custom fallbacks.
116 
117 {% callout warning %}
118 ##### Third-party libraries
119 
120 **Bootstrap does not officially support third-party JavaScript libraries** like Prototype or jQuery UI. Despite `.noConflict` and namespaced events, there may be compatibility problems that you need to fix on your own.
121 {% endcallout %}

123 ## Util
124 
125 All Bootstrap's JavaScript files depend on `util.js` and it has to be included alongside the other JavaScript files. If you're using the compiled (or minified) `bootstrap.js`, there is no need to include this—it's already there.
126 
127 `util.js` includes utility functions and a basic helper for `transitionEnd` events as well as a CSS transition emulator. It's used by the other plugins to check for CSS transition support and to catch hanging transitions.
