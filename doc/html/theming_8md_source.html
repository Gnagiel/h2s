10 ## Introduction
11 
12 In Bootstrap 3, theming was largely driven by variable overrides in LESS, custom CSS, and a separate theme stylesheet that we included in our `dist` files. With some effort, one could completely redesign the look of Bootstrap 3 without touching the core files. Bootstrap 4 provides a familiar, but slightly different approach.
13 
14 Now, theming is accomplished by Sass variables, Sass maps, and custom CSS. There's no more dedicated theme stylesheet; instead, you can enable the built-in theme to add gradients, shadows, and more.
15 
16 ## Sass
17 
18 Utilize our source Sass files to take advantage of variables, maps, mixins, and more.
19 
20 ### File structure
21 
22 Whenever possible, avoid modifying Bootstrap's core files. For Sass, that means creating your own stylesheet that imports Bootstrap so you can modify and extend it. Assuming you're using a package manager like npm, you'll have a file structure that looks like this:
23 
24 {% highlight plaintext %}
25 your-project/
26 ├── scss
27 │ └── custom.scss
28 └── node_modules/
29  └── bootstrap
30  ├── js
31  └── scss
32 {% endhighlight %}
33 
34 If you've downloaded our source files and aren't using a package manager, you'll want to manually setup something similar to that structure, keeping Bootstrap's source files separate from your own.
35 
36 {% highlight plaintext %}
37 your-project/
38 ├── scss
39 │ └── custom.scss
40 └── bootstrap/
41  ├── js
42  └── scss
43 {% endhighlight %}
44 
45 ### Importing
46 
47 In your `custom.scss`, you'll import Bootstrap's source Sass files. You have two options: include all of Bootstrap, or pick the parts you need. We encourage the latter, though be aware there are some requirements and dependencies across our components. You also will need to include some JavaScript for our plugins.
48 
49 {% highlight scss %}
50 // Custom.scss
51 // Option A: Include all of Bootstrap
52 
53 @import "node_modules/bootstrap/scss/bootstrap";
54 {% endhighlight %}
55 
56 {% highlight scss %}
57 // Custom.scss
58 // Option B: Include parts of Bootstrap
59 
60 // Required
61 @import "node_modules/bootstrap/scss/functions";
62 @import "node_modules/bootstrap/scss/variables";
63 @import "node_modules/bootstrap/scss/mixins";
64 
65 // Optional
66 @import "node_modules/bootstrap/scss/reboot";
67 @import "node_modules/bootstrap/scss/type";
68 @import "node_modules/bootstrap/scss/images";
69 @import "node_modules/bootstrap/scss/code";
70 @import "node_modules/bootstrap/scss/grid";
71 {% endhighlight %}
72 
73 With that setup in place, you can begin to modify any of the Sass variables and maps in your `custom.scss`. You can also start to add parts of Bootstrap under the `// Optional` section as needed. We suggest using the full import stack from our `bootstrap.scss` file as your starting point.
74 
75 ### Variable defaults
76 
77 Every Sass variable in Bootstrap 4 includes the `!default` flag allowing you to override the variable's default value in your own Sass without modifying Bootstrap's source code. Copy and paste variables as needed, modify their values, and remove the `!default` flag. If a variable has already been assigned, then it won't be re-assigned by the default values in Bootstrap.
78 
79 Variable overrides within the same Sass file can come before or after the default variables. However, when overriding across Sass files, your overrides must come before you import Bootstrap's Sass files.
80 
81 Here's an example that changes the `background-color` and `color` for the `<body>` when importing and compiling Bootstrap via npm:
82 
83 {% highlight scss %}
84 // Your variable overrides
85 $body-bg: #000;
86 $body-color: #111;
87 
88 // Bootstrap and its default variables
89 @import "node_modules/bootstrap/scss/bootstrap";
90 {% endhighlight %}
91 
92 Repeat as necessary for any variable in Bootstrap, including the global options below.
93 
94 ### Maps and loops
95 
96 Bootstrap 4 includes a handful of Sass maps, key value pairs that make it easier to generate families of related CSS. We use Sass maps for our colors, grid breakpoints, and more. Just like Sass variables, all Sass maps include the `!default` flag and can be overridden and extended.
97 
98 Some of our Sass maps are merged into empty ones by default. This is done to allow easy expansion of a given Sass map, but comes at the cost of making _removing_ items from a map slightly more difficult.
99 
100 #### Modify map
101 
102 To modify an existing color in our `$theme-colors` map, add the following to your custom Sass file:
103 
104 {% highlight scss %}
105 $theme-colors: (
106  "primary": #0074d9,
107  "danger": #ff4136
108 );
109 {% endhighlight %}
110 
111 #### Add to map
112 
113 To add a new color to `$theme-colors`, add the new key and value:
114 
115 {% highlight scss %}
116 $theme-colors: (
117  "custom-color": #900
118 );
119 {% endhighlight %}
120 
121 #### Remove from map
122 
123 To remove colors from `$theme-colors`, or any other map, use `map-remove`:
124 
125 {% highlight scss %}
126 $theme-colors: map-remove($theme-colors, "success", "info", "danger");
127 {% endhighlight %}
128 
129 #### Required keys
130 
131 Bootstrap assumes the presence of some specific keys within Sass maps as we used and extend these ourselves. As you customize the included maps, you may encounter errors where a specific Sass map's key is being used.
132 
133 For example, we use the `primary`, `success`, and `danger` keys from `$theme-colors` for links, buttons, and form states. Replacing the values of these keys should present no issues, but removing them may cause Sass compilation issues. In these instances, you'll need to modify the Sass code that makes use of those values.
134 
135 ### Functions
136 
137 Bootstrap utilizes several Sass functions, but only a subset are applicable to general theming. We've included three functions for getting values from the color maps:
138 
139 {% highlight scss %}
140 @function color($key: "blue") {
141  @return map-get($colors, $key);
142 }
143 
144 @function theme-color($key: "primary") {
145  @return map-get($theme-colors, $key);
146 }
147 
148 @function gray($key: "100") {
149  @return map-get($grays, $key);
150 }
151 {% endhighlight %}
152 
153 These allow you to pick one color from a Sass map much like how you'd use a color variable from v3.
154 
155 {% highlight scss %}
156 .custom-element {
157  color: gray("100");
158  background-color: theme-color("dark");
159 }
160 {% endhighlight %}
161 
162 We also have another function for getting a particular _level_ of color from the `$theme-colors` map. Negative level values will lighten the color, while higher levels will darken.
163 
164 {% highlight scss %}
165 @function theme-color-level($color-name: "primary", $level: 0) {
166  $color: theme-color($color-name);
167  $color-base: if($level > 0, #000, #fff);
168  $level: abs($level);
169 
170  @return mix($color-base, $color, $level * $theme-color-interval);
171 }
172 {% endhighlight %}
173 
174 In practice, you'd call the function and pass in two parameters: the name of the color from `$theme-colors` (e.g., primary or danger) and a numeric level.
175 
176 {% highlight scss %}
177 .custom-element {
178  color: theme-color-level(primary, -10);
179 }
180 {% endhighlight %}
181 
182 Additional functions could be added in the future or your own custom Sass to create level functions for additional Sass maps, or even a generic one if you wanted to be more verbose.
183 
184 ### Color contrast
185 
186 One additional function we include in Bootstrap is the color contrast function, `color-yiq`. It utilizes the [YIQ color space](https://en.wikipedia.org/wiki/YIQ) to automatically return a light (`#fff`) or dark (`#111`) contrast color based on the specified base color. This function is especially useful for mixins or loops where you're generating multiple classes.
187 
188 For example, to generate color swatches from our `$theme-colors` map:
189 
190 {% highlight scss %}
191 @each $color, $value in $theme-colors {
192  .swatch-#{$color} {
193  color: color-yiq($value);
194  }
195 }
196 {% endhighlight %}
197 
198 It can also be used for one-off contrast needs:
199 
200 {% highlight scss %}
201 .custom-element {
202  color: color-yiq(#000); // returns `color: #fff`
203 }
204 {% endhighlight %}
205 
206 You can also specify a base color with our color map functions:
207 
208 {% highlight scss %}
209 .custom-element {
210  color: color-yiq(theme-color("dark")); // returns `color: #fff`
211 }
212 {% endhighlight %}

214 ## Sass options
215 
216 Customize Bootstrap 4 with our built-in custom variables file and easily toggle global CSS preferences with new `$enable-*` Sass variables. Override a variable's value and recompile with `npm run test` as needed.
217 
218 You can find and customize these variables for key global options in our `_variables.scss` file.
219 
220 | Variable | Values | Description |
221 | --------------------------- | ---------------------------------- | -------------------------------------------------------------------------------------- |
222 | `$spacer` | `1rem` (default), or any value > 0 | Specifies the default spacer value to programmatically generate our [spacer utilities]({{ site.baseurl }}/docs/{{ site.docs_version }}/utilities/spacing/). |
223 | `$enable-rounded` | `true` (default) or `false` | Enables predefined `border-radius` styles on various components. |
224 | `$enable-shadows` | `true` or `false` (default) | Enables predefined `box-shadow` styles on various components. |
225 | `$enable-gradients` | `true` or `false` (default) | Enables predefined gradients via `background-image` styles on various components. |
226 | `$enable-transitions` | `true` (default) or `false` | Enables predefined `transition`s on various components. |
227 | `$enable-hover-media-query` | `true` or `false` (default) | **Deprecated** |
228 | `$enable-grid-classes` | `true` (default) or `false` | Enables the generation of CSS classes for the grid system (e.g., `.container`, `.row`, `.col-md-1`, etc.). |
229 | `$enable-caret` | `true` (default) or `false` | Enables pseudo element caret on `.dropdown-toggle`. |
230 | `$enable-print-styles` | `true` (default) or `false` | Enables styles for optimizing printing. |

232 ## Color
233 
234 Many of Bootstrap's various components and utilities are built through a series of colors defined in a Sass map. This map can be looped over in Sass to quickly generate a series of rulesets.
235 
236 ### All colors
237 
238 All colors available in Bootstrap 4, are available as Sass variables and a Sass map in our `scss/_variables.scss` file. This will be expanded upon in subsequent minor releases to add additional shades, much like the [grayscale palette](#grays) we already include.
239 
240 <div class="row">
241  {% for color in site.data.colors %}
242  {% unless color.name == "white" or color.name == "gray" or color.name == "gray-dark" %}
243  <div class="col-md-4">
244  <div class="p-3 mb-3 swatch-{{ color.name }}">{{ color.name | capitalize }}</div>
245  </div>
246  {% endunless %}
247  {% endfor %}
248 </div>
249 
250 Here's how you can use these in your Sass:
251 
252 {% highlight scss %}
253 // With variable
254 .alpha { color: $purple; }
255 
256 // From the Sass map with our `color()` function
257 .beta { color: color("purple"); }
258 {% endhighlight %}
259 
260 [Color utility classes]({{ site.baseurl }}/docs/{{ site.docs_version }}/utilities/colors/) are also available for setting `color` and `background-color`.
261 
262 {% callout info %}
263 In the future, we'll aim to provide Sass maps and variables for shades of each color as we've done with the grayscale colors below.
264 {% endcallout %}

291 
292 {% highlight scss %}
293 $colors: (
294  "blue": $blue,
295  "indigo": $indigo,
296  "purple": $purple,
297  "pink": $pink,
298  "red": $red,
299  "orange": $orange,
300  "yellow": $yellow,
301  "green": $green,
302  "teal": $teal,
303  "cyan": $cyan,
304  "white": $white,
305  "gray": $gray-600,
306  "gray-dark": $gray-800
307 ) !default;

Future updates will strive to improve upon this. Until then, plan on making use of the `${color}` variables and this Sass map.

312 ## Components
313 
314 Many of Bootstrap's components and utilities are built with `@each` loops that iterate over a Sass map. This is especially helpful for generating variants of a component by our `$theme-colors` and creating responsive variants for each breakpoint. As you customize these Sass maps and recompile, you'll automatically see your changes reflected in these loops.
315 
316 ### Modifiers
317 
318 Many of Bootstrap's components are built with a base-modifier class approach. This means the bulk of the styling is contained to a base class (e.g., `.btn`) while style variations are confined to modifier classes (e.g., `.btn-danger`). These modifier classes are built from the `$theme-colors` map to make customizing the number and name of our modifier classes.
319 
320 Here are two examples of how we loop over the `$theme-colors` map to generate modifiers to the `.alert` component and all our `.bg-*` background utilities.
321 
322 {% highlight scss %}
323 // Generate alert modifier classes
324 @each $color, $value in $theme-colors {
325  .alert-#{$color} {
326  @include alert-variant(theme-color-level($color, -10), theme-color-level($color, -9), theme-color-level($color, 6));
327  }
328 }
329 
330 // Generate `.bg-*` color utilities
331 @each $color, $value in $theme-colors {
332  @include bg-variant('.bg-#{$color}', $value);
333 }
334 {% endhighlight %}
335 
336 ### Responsive
337 
338 These Sass loops aren't limited to color maps, either. You can also generate responsive variations of your components or utilities. Take for example our responsive text alignment utilities where we mix an `@each` loop for the `$grid-breakpoints` Sass map with a media query include.
339 
340 {% highlight scss %}
341 @each $breakpoint in map-keys($grid-breakpoints) {
342  @include media-breakpoint-up($breakpoint) {
343  $infix: breakpoint-infix($breakpoint, $grid-breakpoints);
344 
345  .text#{$infix}-left { text-align: left !important; }
346  .text#{$infix}-right { text-align: right !important; }
347  .text#{$infix}-center { text-align: center !important; }
348  }
349 }
350 {% endhighlight %}
351 
352 Should you need to modify your `$grid-breakpoints`, your changes will apply to all the loops iterating over that map.

354 ## CSS variables
355 
356 Bootstrap 4 includes around two dozen [CSS custom properties (variables)](https://developer.mozilla.org/en-US/docs/Web/CSS/Using_CSS_variables) in it's compiled CSS. These provide easy access to commonly used values like our theme colors, breakpoints, and primary font stacks when working in your browser's Inspector, a code sandbox, or general prototyping.

395 ### Examples
396 
397 CSS variables offer similar flexibility to Sass's variables, but without the need for compilation before being served to the browser. For example, here we're resetting our page's font and link styles with CSS variables.
398

404  color: var(--blue);
