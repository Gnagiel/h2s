9 ## How it works
10 
11 Scrollspy has a few requirements to function properly:
12 
13 - If you're building our JavaScript from source, it [requires `util.js`]({{ site.baseurl }}/docs/{{ site.docs_version }}/getting-started/javascript/#util).
14 - It must be used on a Bootstrap [nav component]({{ site.baseurl }}/docs/{{ site.docs_version }}/components/navs/) or [list group]({{ site.baseurl }}/docs/{{ site.docs_version }}/components/list-group/).
15 - Scrollspy requires `position: relative;` on the element you're spying on, usually the `<body>`.
16 - When spying on elements other than the `<body>`, be sure to have a `height` set and `overflow-y: scroll;` applied.
17 - Anchors (`<a>`) are required and must point to an element with that `id`.
18 
19 When successfully implemented, your nav or list group will update accordingly, moving the `.active` class from one item to the next based on their associated targets.

227 ## Usage
228 
229 ### Via data attributes
230 
231 To easily add scrollspy behavior to your topbar navigation, add `data-spy="scroll"` to the element you want to spy on (most typically this would be the `<body>`). Then add the `data-target` attribute with the ID or class of the parent element of any Bootstrap `.nav` component.
232 
233 {% highlight css %}
234 body {
235  position: relative;
236 }
237 {% endhighlight %}
238 
239 {% highlight html %}
240 <body data-spy="scroll" data-target="#navbar-example">
241  ...
242  <div id="navbar-example">
243  <ul class="nav nav-tabs" role="tablist">
244  ...
245  </ul>
246  </div>
247  ...
248 </body>
249 {% endhighlight %}
250 
251 ### Via JavaScript
252 
253 After adding `position: relative;` in your CSS, call the scrollspy via JavaScript:
254 
255 {% highlight js %}
256 $('body').scrollspy({ target: '#navbar-example' })
257 {% endhighlight %}
258 
259 {% callout danger %}
260 #### Resolvable ID targets required
261 
262 Navbar links must have resolvable id targets. For example, a `<a href="#home">home</a>` must correspond to something in the DOM like `<div id="home"></div>`.
263 {% endcallout %}
264 
265 {% callout info %}
266 #### Non-`:visible` target elements ignored
267 
268 Target elements that are not [`:visible` according to jQuery](https://api.jquery.com/visible-selector/) will be ignored and their corresponding nav items will never be highlighted.
269 {% endcallout %}
270 
271 ### Methods
272 
273 #### `.scrollspy('refresh')`
274 
275 When using scrollspy in conjunction with adding or removing of elements from the DOM, you'll need to call the refresh method like so:
276 
277 {% highlight js %}
278 $('[data-spy="scroll"]').each(function () {
279  var $spy = $(this).scrollspy('refresh')
280 })
281 {% endhighlight %}
282 
283 #### `.scrollspy('dispose')`
284 
285 Destroys an element's scrollspy.
286 
287 ### Options
288 
289 Options can be passed via data attributes or JavaScript. For data attributes, append the option name to `data-`, as in `data-offset=""`.
