9 ## Global settings
10 
11 Bootstrap sets basic global display, typography, and link styles. When more control is needed, check out the [textual utility classes]({{ site.baseurl }}/docs/{{ site.docs_version }}/utilities/text/).
12 
13 - Use a [native font stack]({{ site.baseurl }}/docs/{{ site.docs_version }}/content/reboot/#native-font-stack) that selects the best `font-family` for each OS and device.
14 - For a more inclusive and accessible type scale, we assume the browser default root `font-size` (typically 16px) so visitors can customize their browser defaults as needed.
15 - Use the `$font-family-base`, `$font-size-base`, and `$line-height-base` attributes as our typographic base applied to the `<body>`.
16 - Set the global link color via `$link-color` and apply link underlines only on `:hover`.
17 - Use `$body-bg` to set a `background-color` on the `<body>` (`#fff` by default).
18 
19 These styles can be found within `_reboot.scss`, and the global variables are defined in `_variables.scss`. Make sure to set `$font-size-base` in `rem`.

110 ## Display headings
111 
112 Traditional heading elements are designed to work best in the meat of your page content. When you need a heading to stand out, consider using a **display heading**—a larger, slightly more opinionated heading style.

150 ## Inline text elements
151 
152 Styling for common inline HTML5 elements.
153 
154 {% example html %}
155 <p>You can use the mark tag to <mark>highlight</mark> text.</p>
156 <p><del>This line of text is meant to be treated as deleted text.</del></p>
157 <p><s>This line of text is meant to be treated as no longer accurate.</s></p>
158 <p><ins>This line of text is meant to be treated as an addition to the document.</ins></p>
159 <p><u>This line of text will render as underlined</u></p>
160 <p><small>This line of text is meant to be treated as fine print.</small></p>
161 <p><strong>This line rendered as bold text.</strong></p>
162 <p><em>This line rendered as italicized text.</em></p>
163 {% endexample %}
164 
165 `.mark` and `.small` classes are also available to apply the same styles as `<mark>` and `<small>` while avoiding any unwanted semantic implications that the tags would bring.
166 
167 While not shown above, feel free to use `<b>` and `<i>` in HTML5. `<b>` is meant to highlight words or phrases without conveying additional importance while `<i>` is mostly for voice, technical terms, etc.

173 ## Abbreviations
174 
175 Stylized implementation of HTML's `<abbr>` element for abbreviations and acronyms to show the expanded version on hover. Abbreviations have a default underline and gain a help cursor to provide additional context on hover and to users of assistive technologies.
176 
177 Add `.initialism` to an abbreviation for a slightly smaller font-size.
178 
179 {% example html %}
180 <p><abbr title="attribute">attr</abbr></p>
181 <p><abbr title="HyperText Markup Language" class="initialism">HTML</abbr></p>
182 {% endexample %}

184 ## Blockquotes
185 
186 For quoting blocks of content from another source within your document. Wrap `<blockquote class="blockquote">` around any <abbr title="HyperText Markup Language">HTML</abbr> as the quote.
187 
188 {% example html %}
189 <blockquote class="blockquote">
190  <p class="mb-0">Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer posuere erat a ante.</p>
191 </blockquote>
192 {% endexample %}
193 
194 ### Naming a source
195 
196 Add a `<footer class="blockquote-footer">` for identifying the source. Wrap the name of the source work in `<cite>`.
197 
198 {% example html %}
199 <blockquote class="blockquote">
200  <p class="mb-0">Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer posuere erat a ante.</p>
201  <footer class="blockquote-footer">Someone famous in <cite title="Source Title">Source Title</cite></footer>
202 </blockquote>
203 {% endexample %}

262 
263 Align terms and descriptions horizontally by using our grid system's predefined classes (or semantic mixins). For longer terms, you can optionally add a `.text-truncate` class to truncate the text with an ellipsis.

267  <dt class="col-sm-3">Description lists</dt>

269 
270  <dt class="col-sm-3">Euismod</dt>
271  <dd class="col-sm-9">
272  <p>Vestibulum id ligula porta felis euismod semper eget lacinia odio sem nec elit.</p>
273  <p>Donec id elit non mi porta gravida at eget metus.</p>
274  </dd>
275 
276  <dt class="col-sm-3">Malesuada porta</dt>
277  <dd class="col-sm-9">Etiam porta sem malesuada magna mollis euismod.</dd>

279  <dt class="col-sm-3 text-truncate">Truncated term is truncated</dt>

281 
282  <dt class="col-sm-3">Nesting</dt>
283  <dd class="col-sm-9">
284  <dl class="row">

288  </dd>

292 ## Responsive typography
293 
294 *Responsive typography* refers to scaling text and components by simply adjusting the root element's `font-size` within a series of media queries. Bootstrap doesn't do this for you, but it's fairly easy to add if you need it.
295 
296 Here's an example of it in practice. Choose whatever `font-size`s and media queries you wish.
297 
298 {% highlight scss %}
299 html {
300  font-size: 1rem;
301 }
302 
303 @include media-breakpoint-up(sm) {
304  html {
305  font-size: 1.2rem;
306  }
307 }
308 
309 @include media-breakpoint-up(md) {
310  html {
311  font-size: 1.4rem;
312  }
313 }
314 
315 @include media-breakpoint-up(lg) {
316  html {
317  font-size: 1.6rem;
318  }
319 }
320 {% endhighlight %}
