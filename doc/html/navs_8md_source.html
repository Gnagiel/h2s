9 ## Base nav
10 
11 Navigation available in Bootstrap share general markup and styles, from the base `.nav` class to the active and disabled states. Swap modifier classes to switch between each style.
12 
13 The base `.nav` component is built with flexbox and provide a strong foundation for building all types of navigation components. It includes some style overrides (for working with lists), some link padding for larger hit areas, and basic disabled styling.
14 
15 {% callout info %}
16 The base `.nav` component does not include any `.active` state. The following examples include the class, mainly to demonstrate that this particular class does not trigger any special styling.
17 {% endcallout %}
18 
19 {% example html %}
20 <ul class="nav">
21  <li class="nav-item">
22  <a class="nav-link active" href="#">Active</a>
23  </li>
24  <li class="nav-item">
25  <a class="nav-link" href="#">Link</a>
26  </li>
27  <li class="nav-item">
28  <a class="nav-link" href="#">Link</a>
29  </li>
30  <li class="nav-item">
31  <a class="nav-link disabled" href="#">Disabled</a>
32  </li>
33 </ul>
34 {% endexample %}
35 
36 Classes are used throughout, so your markup can be super flexible. Use `<ul>`s like above, or roll your own with say a `<nav>` element. Because the `.nav` uses `display: flex`, the nav links behave the same as nav items would, but without the extra markup.
37 
38 {% example html %}
39 <nav class="nav">
40  <a class="nav-link active" href="#">Active</a>
41  <a class="nav-link" href="#">Link</a>
42  <a class="nav-link" href="#">Link</a>
43  <a class="nav-link disabled" href="#">Disabled</a>
44 </nav>
45 {% endexample %}

167 ### Fill and justify
168 
169 Force your `.nav`'s contents to extend the full available width one of two modifier classes. To proportionately fill all available space with your `.nav-item`s, use `.nav-fill`. Notice that all horizontal space is occupied, but not every nav item has the same width.
170 
171 {% example html %}
172 <ul class="nav nav-pills nav-fill">
173  <li class="nav-item">
174  <a class="nav-link active" href="#">Active</a>
175  </li>
176  <li class="nav-item">
177  <a class="nav-link" href="#">Longer nav link</a>
178  </li>
179  <li class="nav-item">
180  <a class="nav-link" href="#">Link</a>
181  </li>
182  <li class="nav-item">
183  <a class="nav-link disabled" href="#">Disabled</a>
184  </li>
185 </ul>
186 {% endexample %}
187 
188 When using a `<nav>`-based navigation, be sure to include `.nav-item` on the anchors.
189 
190 {% example html %}
191 <nav class="nav nav-pills nav-fill">
192  <a class="nav-item nav-link active" href="#">Active</a>
193  <a class="nav-item nav-link" href="#">Link</a>
194  <a class="nav-item nav-link" href="#">Link</a>
195  <a class="nav-item nav-link disabled" href="#">Disabled</a>
196 </nav>
197 {% endexample %}
198 
199 For equal-width elements, use `.nav-justified`. All horizontal space will be occupied by nav links, but unlike the `.nav-fill` above, every nav item will be the same width.
200 
201 {% example html %}
202 <nav class="nav nav-pills nav-justified">
203  <a class="nav-link active" href="#">Active</a>
204  <a class="nav-link" href="#">Longer nav link</a>
205  <a class="nav-link" href="#">Link</a>
206  <a class="nav-link disabled" href="#">Disabled</a>
207 </nav>
208 {% endexample %}
209 
210 Similar to the `.nav-fill` example using a `<nav>`-based navigation, be sure to include `.nav-item` on the anchors.
211 
212 {% example html %}
213 <nav class="nav nav-pills nav-justified">
214  <a class="nav-item nav-link active" href="#">Active</a>

221 ## Working with flex utilities
222 
223 If you need responsive nav variations, consider using a series of [flexbox utilities]({{ site.baseurl }}/docs/{{ site.docs_version }}/utilities/flex/). While more verbose, these utilities offer greater customization across responsive breakpoints. In the example below, our nav will be stacked on the lowest breakpoint, then adapt to a horizontal layout that fills the available width starting from the small breakpoint.
224 
225 {% example html %}
226 <nav class="nav nav-pills flex-column flex-sm-row">
227  <a class="flex-sm-fill text-sm-center nav-link active" href="#">Active</a>
228  <a class="flex-sm-fill text-sm-center nav-link" href="#">Link</a>
229  <a class="flex-sm-fill text-sm-center nav-link" href="#">Link</a>
230  <a class="flex-sm-fill text-sm-center nav-link disabled" href="#">Disabled</a>
231 </nav>
232 {% endexample %}

234 ## Regarding accessibility
235 
236 If you're using navs to provide a navigation bar, be sure to add a `role="navigation"` to the most logical parent container of the `<ul>`, or wrap a `<nav>` element around the whole navigation. Do not add the role to the `<ul>` itself, as this would prevent it from being announced as an actual list by assistive technologies.
237 
238 Note that navigation bars, even if visually styled as tabs with the `.nav-tabs` class, should **not** be given `role="tablist"`, `role="tab"` or `role="tabpanel"` attributes. These are only appropriate for dynamic tabbed interfaces, as described in the [<abbr title="Web Accessibility Initiative">WAI</abbr> <abbr title="Accessible Rich Internet Applications">ARIA</abbr> Authoring Practices](https://www.w3.org/TR/wai-aria-practices/#tabpanel). See [JavaScript behavior](#javascript-behavior) for dynamic tabbed interfaces in this section for an example.

296 ## JavaScript behavior
297 
298 Use the tab JavaScript plugin—include it individually or through the compiled `bootstrap.js` file—to extend our navigational tabs and pills to create tabbable panes of local content, even via dropdown menus.
299 
300 If you're building our JavaScript from source, it [requires `util.js`]({{ site.baseurl }}/docs/{{ site.docs_version }}/getting-started/javascript/#util).
301 
302 Dynamic tabbed interfaces, as described in the [<abbr title="Web Accessibility Initiative">WAI</abbr> <abbr title="Accessible Rich Internet Applications">ARIA</abbr> Authoring Practices](https://www.w3.org/TR/wai-aria-practices/#tabpanel), require `role="tablist"`, `role="tab"`, `role="tabpanel"`, and additional `aria-` attributes in order to convey their structure, functionality and current state to users of assistive technologies (such as screen readers).
303 
304 Note that dynamic tabbed interfaces should <em>not</em> contain dropdown menus, as this causes both usability and accessibility issues. From a usability perspective, the fact that the currently displayed tab's trigger element is not immediately visible (as it's inside the closed dropdown menu) can cause confusion. From an accessibility point of view, there is currently no sensible way to map this sort of construct to a standard WAI ARIA pattern, meaning that it cannot be easily made understandable to users of assistive technologies.

510 ### Via JavaScript
511 
512 Enable tabbable tabs via JavaScript (each tab needs to be activated individually):
513 
514 {% highlight js %}
515 $('#myTab a').on('click', function (e) {
516  e.preventDefault()
517  $(this).tab('show')
518 })
519 {% endhighlight %}
520 
521 You can activate individual tabs in several ways:
522 
523 {% highlight js %}
524 $('#myTab a[href="#profile"]').tab('show') // Select tab by name
525 $('#myTab li:first-child a').tab('show') // Select first tab
526 $('#myTab li:last-child a').tab('show') // Select last tab
527 $('#myTab li:nth-child(3) a').tab('show') // Select third tab
528 {% endhighlight %}

582 #### .tab('show')
583 
584 Selects the given tab and shows its associated pane. Any other tab that was previously selected becomes unselected and its associated pane is hidden. **Returns to the caller before the tab pane has actually been shown** (i.e. before the `shown.bs.tab` event occurs).
585

587 $('#someTab').tab('show')

594 ### Events
595 
596 When showing a new tab, the events fire in the following order:
597 
598 1. `hide.bs.tab` (on the current active tab)
599 2. `show.bs.tab` (on the to-be-shown tab)
600 3. `hidden.bs.tab` (on the previous active tab, the same one as for the `hide.bs.tab` event)
601 4. `shown.bs.tab` (on the newly-active just-shown tab, the same one as for the `show.bs.tab` event)
602 
603 If no tab was already active, then the `hide.bs.tab` and `hidden.bs.tab` events will not be fired.
604 
605 <table class="table table-bordered table-striped">
606  <thead>
607  <tr>
608  <th style="width: 150px;">Event Type</th>
609  <th>Description</th>
610  </tr>
611  </thead>
612  <tbody>
613  <tr>
614  <td>show.bs.tab</td>
615  <td>This event fires on tab show, but before the new tab has been shown. Use <code>event.target</code> and <code>event.relatedTarget</code> to target the active tab and the previous active tab (if available) respectively.</td>
616  </tr>
617  <tr>
618  <td>shown.bs.tab</td>
619  <td>This event fires on tab show after a tab has been shown. Use <code>event.target</code> and <code>event.relatedTarget</code> to target the active tab and the previous active tab (if available) respectively.</td>
620  </tr>
621  <tr>
622  <td>hide.bs.tab</td>
623  <td>This event fires when a new tab is to be shown (and thus the previous active tab is to be hidden). Use <code>event.target</code> and <code>event.relatedTarget</code> to target the current active tab and the new soon-to-be-active tab, respectively.</td>
624  </tr>
625  <tr>
626  <td>hidden.bs.tab</td>
627  <td>This event fires after a new tab is shown (and thus the previous active tab is hidden). Use <code>event.target</code> and <code>event.relatedTarget</code> to target the previous active tab and the new active tab, respectively.</td>
628  </tr>
629  </tbody>
630 </table>
631 
632 {% highlight js %}
633 $('a[data-toggle="tab"]').on('shown.bs.tab', function (e) {
634  e.target // newly activated tab
635  e.relatedTarget // previous active tab
636 })
637 {% endhighlight %}
